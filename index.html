📘 需求说明 · 代理渠道延保分润系统

本页面向产品、研发、测试与业务方说明本原型覆盖的需求边界与交付要点。详细字段与验收以 PRD 为准；下方为结构化导读，点击顶部「角色登录」可进入各端界面演示。

一句话：围绕客车 ETC / 货车 ETC / 记录仪三类产品的进销存、激活/绑定、分润计费与提现，为主管、销售、一级代理提供小程序能力，并为运营提供 Web 后台（渠道、门店、分润配置、审核与权限隔离）。一级代理提现须认证/签约闭环；分润入账及提现审核通过 / 审核未通过 / 打款到账均触发微信订阅消息；后台提现审核与之一致校验。

建设目标与价值

分润：提升代理商积极性，让代理清晰看到每笔收益来源，形成正向激励。
进销存可视化：即时获取代理商设备数据情况，做到库存剩余较少时及时沟通补货，降低因设备断货造成的业务损失。
  • 分润透明：各角色按权限查看分润流水、账户余额与提现进度；流水区分产品类型与触发方式（如货车 ETC 须展示车型与对应分润金额）。
  • 进销存可视：按产品与代理维度展示进货、激活/绑定、待处理；支持按激活率（或绑定率）排序。
  • 风险可控：渠道信息变更、分润配置等走审核；提现对接灵工平台；权限边界在「权限对比」页可查。
  • 提现与触达：一级代理须完成「我的 — 认证/签约」后方可申请提现；运营在「提现审核」点击通过前须校验代理已认证且已签约（原型与演示数据联动）。微信订阅消息：① 分润入账实时推送「新分润入账」（金额、产品类型）；② 提现审核通过后推送（申请金额、通过时间等）；③ 提现审核未通过（驳回）后推送（申请金额、驳回原因、时间）；④ 灵工打款到账成功后推送「提现到账」（到账金额、时间）。各类均需代理提前授权对应订阅模板。

核心业务规则摘要

主题规则要点
分润触发记录仪：智能服务订单；货车 ETC：设备激活（流水须含车型与实发分润）；客车 ETC：仅延保触发（设备激活不触发，流水标签「客车ETC-延保」）。
流水与汇总列表不展示设备 SN；不展示「退款补偿」类条目。一级代理「计费」本月汇总中客车 ETC 为延保笔数口径；流水列表仍可含客车激活等多类型演示行。
进销存主管/销售代理明细支持按产品筛选下的激活率或绑定率排序，默认从高到低。
一级代理进货批次不展示 SN、不展示分润比例；提现须先完成「认证/签约」；微信订阅消息含：① 分润入账；② 提现审核通过；③ 提现审核未通过；④ 提现到账（均需授权模板）。
渠道配置弹窗必填产品类型（三类多选）；可选「是否展示进销存」；货车分润按车型分档单笔金额（演示单项≤500 元）；销售人员模块与列表字段对齐。
门店后台门店管理以截图还原列表；操作「生成专属二维码」绑定渠道+门店。
提现审核运营「通过」前校验代理已认证且已签约；未完成则拦截（原型中与一级代理「我的 — 认证/签约」演示状态联动，如黄石新捷顺名车）。
后台订单退款三分支判断：不涉及分润→直接退；涉及分润未提现→直接退+自动扣减代理余额+小程序「退款扣回」流水；涉及分润已提现→弹窗线下通知代理，详见「退款页面」。

💡 分润归因基础逻辑

三类产品的分润归因方式完全不同，开发需按产品类型走各自归因链路。

产品归因标识规则说明
记录仪 IMEI 用户购买智能服务订单时，系统通过设备 IMEI 查找所属渠道，命中则触发分润
客车ETC SN 延保订单支付成功后，系统通过设备 SN 号 查找所属渠道，命中则按延保金额×分润比例入账
货车ETC 渠道码 设备激活时须扫渠道码才能绑定渠道，未扫码则不参与分润；渠道码绑定在该渠道下各门店身上，每个门店有专属码，由后台「门店管理」维护；激活后系统按码对应渠道及车型分档金额入账
⚠️ 货车ETC 特别注意：渠道码是唯一归因凭证；若门店无专属码或用户未扫码，系统无法识别渠道归属，该笔激活不产生分润，且无法补录。运营须确保每个门店均已在后台生成并下发专属渠道码。

🗺️ 主体业务路程

1
运营录入 SN / 配置渠道
后台「渠道管理」新增渠道（含产品类型、分润比例、归属销售）；客车ETC / 记录仪绑定 SN 号段至渠道；货车ETC 在「门店管理」为每个门店生成专属渠道码并下发给门店
2
代理 / 用户触发分润事件
客车ETC 车主办延保（19.9/29.9/59.9元），SN 命中渠道 → 按延保金额×比例入账
货车ETC 用户扫门店专属渠道码激活设备，按车型（轻卡/重卡/牵引）分档固定金额入账
记录仪 用户购买智能服务订单，IMEI 命中渠道 → 按订单金额×比例入账
3
分润自动入账 · 代理实时可见
分润引擎计算后写入代理账户余额，代理在小程序首页和「计费」Tab 实时看到新流水；退款发生时，流水新增红色退款条目（负金额 + 双标签），原条目保留
4
代理完成认证 / 签约（提现前置）
一级代理在「我的 → 认证/签约」提交姓名、身份证、银行卡信息，完成实名认证后跳转电子签约；两项均完成才可申请提现；运营提现审核点击「通过」前系统自动校验此状态
5
代理申请提现 · 运营审核
代理在「账户」Tab 输入提现金额，确认后提交；后台运营在「提现审核」列表审核（核验认证/签约状态）；审核通过后由灵工平台打款，预计 1–3 个工作日到账
6
打款到账 · 微信订阅消息通知
提现提交后：运营审核通过或驳回时分别下发对应微信订阅消息；灵工打款成功后再下发「提现到账」（含到账金额+时间）。另：分润入账时下发「新分润入账」。多类模板均需代理提前授权。

退款判断逻辑（三分支）

情形 判断条件 处理方式

③ 参与分润 · 已提现 订单分润已通过提现申请，并已打款至代理银行账户 不可自动退款；弹提示弹窗，告知运营需线下通知代理商：①代理向用户全额退款；②运营线下将扣除分润后的余额退给代理商；弹窗展示：渠道名、代理姓名、手机号、订单金额、分润金额（已提现）、线下退款金额

小程序同步说明

  • 情形①②退款成功后，退款记录均需实时同步至代理小程序（如延保订单详情、计费Tab）
  • 情形②退款后，代理小程序分润流水增加「↩ 退款扣回」条目：红色背景、负数金额、标签「已退款」+「分润已扣回」
  • 情形②退款后，代理小程序首页/提现页的「可提现余额」需实时刷新（减去对应分润金额）
  • 情形③为纯线下流程，系统不自动变更代理余额，依赖运营与代理线下沟通完成

「未提现」的判断规则

系统需记录每笔分润的「入账时间」与「提现关联关系」。判断该分润是否已被提现的依据：该分润金额所在的结算周期是否已生成「已完成」状态的提现记录。若已存在已打款提现记录，则视为「已提现」，走情形③；否则走情形②。
边界情形：若代理申请了提现但提现审核「待审核/审核中」，暂按「未提现」处理（因未实际到账）。
